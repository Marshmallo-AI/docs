# Introduction

Marlo is the learning layer for agent runs. You keep your agent code and infrastructure. Marlo records what the agent did, scores the outcome, and turns that result into guidance the agent can use later.

## What Marlo Does
Marlo adds a feedback loop on top of your existing agent:
- **Capture** the full timeline of model calls, tool calls, and outputs.
- **Score** each task with a clear rationale.
- **Learn** from the result and store it as guidance.
- **Apply** active learnings to future runs.

## Why It Matters
In production, failures are hard to debug because the evidence is scattered across logs, prompts, and tool outputs. Marlo gives you a single, consistent record of what happened so you can grade outcomes and improve reliably.

## What You Send
Marlo needs a small, stable set of data for every task:
- Agent definition (system prompt, tools, MCP, model config)
- Task input and output
- LLM calls and tool calls
- Task status
