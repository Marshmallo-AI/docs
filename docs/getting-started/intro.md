# Marlo Docs

## What Marlo Does
Marlo wraps your existing agent. It captures what the agent did, scores each task, and turns the results into learnings you can reuse. You keep running your agent. Marlo only collects signals and returns learnings.

Marlo works for:
- a single agent
- a system with sub‑agents
- agent‑to‑agent workflows

## One Sentence Flow
Start session → start task → stream events → end task → reward → learning → reuse.

## Why It Matters
Most agents are black boxes. When something fails, it is hard to know why. Marlo keeps the evidence trail and turns it into learnings, so your system improves over time.

## When to Use It
- You already run an agent in production.
- You want to capture LLM calls, tool calls, reasoning (if available), and outcomes.
- You want reward signals and learnings from real runs.
- You want to guide agent behavior with learnings.
