# Orchestrator

An orchestrator system has one root agent that coordinates the task and delegates work to several specialized agents. The root agent decides which subâ€‘agent to call, collects their outputs, and produces the final response. This is a supported agentic system in Marlo.

![Orchestrator agent system diagram](/img/orchestrator.png)

How Marlo represents it:
- One agent is the entry agent (the orchestrator).
- Each subâ€‘agent is linked to the orchestrator with a parent agent id.
- The trace shows which subâ€‘agent ran, what it returned, and how the orchestrator finished the task.

What you need to send:
- Agent definitions for the orchestrator and every subâ€‘agent.
- Agent start and agent end events for each agent run.
- Parent agent ids that connect subâ€‘agents to the orchestrator.
- Task start and task end events, plus all LLM and tool calls.
